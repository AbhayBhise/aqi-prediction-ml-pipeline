# 🚀 Production Deployment Blueprint: Flask ML API & React Frontend

This guide outlines the professional, step-by-step blueprint to host and deploy the **Agentic AQI Forecasting & Analysis System** on **Render** (or equivalent cloud platforms). 

By separating the project into a **Flask ML API (Web Service)** and a **Vite React UI (Static Site)**, we satisfy modern academic and industry standards for multi-tier microservice architectures.

---

## 📂 Phase 1: Preparation & Git Push

Before deploying, ensure your code is pushed to a remote Git repository (e.g., GitHub).

### 1. Verification of `.gitignore`
Make sure heavy binaries, local cache directories, virtual environments, and private credentials are excluded from the repository.
Your root and subdirectory `.gitignore` files are already configured to exclude:
*   `node_modules/` (Frontend package directories)
*   `__pycache__/` (Python compiled bytecode)
*   `aqi_pa_v2_venv/` (Local Python virtual environment)
*   `.env` (Private API keys)

### 2. Push to GitHub
Open your terminal (PowerShell or Git Bash) in the project root (`AQI PREDICTION VERSION 2/`) and run:
```bash
# Initialize git repository
git init

# Add all files to staging (skipping ignored files)
git add .

# Commit changes
git commit -m "feat: production hardened multi-tier deployment ready"

# Create a new repository on GitHub, then link and push:
git branch -M main
git remote add origin https://github.com/YOUR_GITHUB_USERNAME/YOUR_REPO_NAME.git
git push -u origin main
```

## 🐍 Phase 2: Deploying the Backend (Flask API)

Deploy the Flask backend as a **Web Service** on Render.

### 1. Create Web Service
1. Log in to the [Render Dashboard](https://dashboard.render.com).
2. Click **New +** $\rightarrow$ **Web Service**.
3. Connect your GitHub repository.

### 2. Configure Settings
Configure the following parameters in the creation form:
*   **Name**: `aqi-predictive-api` (or any unique name)
*   **Environment**: `Python 3`
*   **Region**: Select the region closest to you (e.g., `Singapore` or `Oregon`)
*   **Branch**: `main`
*   **Root Directory**: `backend` *(Crucial: This isolates the Python backend workspace)*
*   **Build Command**: `pip install -r requirements.txt`
*   **Start Command**: `gunicorn -w 2 -k gthread --threads 4 api.app:app`
*   **Instance Type**: `Free` (or higher)

### 3. Environment Variables (Secret Files & Keys)
Click the **Environment** tab on Render and add the following keys:
1.  `GOOGLE_API_KEY`: *(Your Google Gemini API Key for the ReAct Agentic Chatbot)*
2.  `OPENWEATHER_API_KEY`: *(Your OpenWeatherMap free tier API key)*
3.  `FRONTEND_URL`: `https://YOUR-FRONTEND-URL.onrender.com` *(The URL of your frontend site, created in Phase 3. You can update this variable later after creating the frontend)*
4.  `PORT`: `10000` *(Render sets this automatically, and Flask-Talisman is pre-configured to adapt to it)*

> [!TIP]
> **What about model sizes and startup limits?**
> The backend implements **Lazy Model Loading** protected by a `threading.Lock()`. This prevents Render's free tier from running out of RAM (limit: 512MB) during startup, ensuring your service initializes in **less than 2 seconds** and safely loads models upon the first forecast request.

## ⚛️ Phase 3: Deploying the Frontend (React / Vite)

Deploy the React user interface as a **Static Site** on Render.

### 1. Create Static Site
1. Go back to your [Render Dashboard](https://dashboard.render.com).
2. Click **New +** $\rightarrow$ **Static Site**.
3. Select your connected GitHub repository.

### 2. Configure Settings
*   **Name**: `aqi-predictive-dashboard` (or any unique name)
*   **Branch**: `main`
*   **Root Directory**: `frontend` *(Crucial: This isolates the React/Vite workspace)*
*   **Build Command**: `npm install && npm run build`
*   **Publish Directory**: `dist` *(Vite outputs static build assets to the dist folder)*

### 3. Environment Variables
Click the **Environment** tab on the Static Site and add:
*   `VITE_API_BASE_URL`: `https://aqi-predictive-api.onrender.com` *(The exact URL generated by Render for your Web Service in Phase 2)*

### 4. Configure Client-Side Routing Rewrite (Crucial Step!)
Since the frontend uses client-side routing (`react-router-dom`), refreshing pages like `/lstm` or `/generative` directly will return a `404 Not Found` from Render's static server.
**How to fix it:**
1. Go to your Static Site's settings in Render.
2. Click **Redirects/Rewrites** in the sidebar.
3. Click **Add Rule**.
4. Configure as follows:
   *   **Source Path**: `/*`
   *   **Destination**: `/index.html`
   *   **Action**: `Rewrite` *(This transparently maps all router sub-paths to Vite's root router, preventing 404s!)*
5. Click **Save Changes**.

## ✅ Phase 4: Post-Deployment Verification Checklist

Once both services show a status of **Live** (green checkmark), run these verification tests:

### 1. Backend Liveness & Readiness Tests
Open a web browser and query the backend health routes:
*   `https://aqi-predictive-api.onrender.com/health`
    *   *Expected output*: `{"status": "healthy"}` (Confirms system liveness)
*   `https://aqi-predictive-api.onrender.com/ready`
    *   *Expected output*: `{"ready": true}` (Confirms dataset preloaded in memory)

### 2. CORS Verification
Visit your frontend static site URL. 
*   If the landing dashboard loads live weather indices, and the comparisons page renders dynamically, CORS is functioning perfectly.
*   *Security Note*: Talisman secures all request headers with `X-Frame-Options`, `X-Content-Type-Options`, and `Referrer-Policy` automatically in production.

### 3. Agentic Chatbot Check
Open the Agentic chatbot page (`/agent`) on your deployed URL and ask:
*   *"What is the current AQI in Bangalore?"*
    *   *Reasoning Trace*: You should see the ReAct thought loop query the Weather API, run prediction algorithms, and render a safety precaution advisory.
